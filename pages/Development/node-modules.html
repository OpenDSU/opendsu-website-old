This page contains an overview of the major PrivateSky modules (public github repositories) and their purpose. Module | Purpose privatesky | This is the start repository for the PrivateSky developers (core team and external contributors) psk-release | The privatesky repository is packed with browserify and PrivateSky specific building tools to create periodic releases. The releases are aimed to make easier for third party companies to develop PrivateSky blockchain domains, CSB constitutions and cloud applications (backends). blockchain | This module contains the PrivateSky blockchain implementation, the OBFT consensus algorithm and various strategies for storing blocks and blockchain world state. This module contains also the implementation of assets or transactions (types of swarm communication) used by smart contracts and by the secret smart contracts. swarm-engine | This module contains the implementation of choreographies (swarm) and of the interaction type of swarm. callflow | This module constitute the base for PrivateSky implementation of the swarm communication programming paradigm. overwrite-require | Utility module that should be loaded first in the PrivateSky ecosistem. psk-security-context | Offer implementation for various security contexts strategies. swarmutils | Utility functionalities used by callflow. cardinal | Cardinal is a mini MVC framework for web-components UI for Self Sovereign Applications psk-bindable-model | Creates observable models for Cardinal. It is dependency free and uses proxies. annotation-engine | Dependencies free modules that can be reused to create annotations to DSLs (Domain Specific Languages) syndicate | Dependencies free module offering pools of threads or isolates pskisolates | Node binding for the isolates technology that enables secure execution of unsecure code. pskcrypto | Wrapper module over node crypto. bar | Dependency free library to work with BARs (Brick Archives) bar-fs-adapter | csb | Portable (node and browsers) Implementation of the CSB (Cloud Safe Boxes) edfs | edfs-brick-storage | virtualmq | Enable communication between PrivateSKy nodes and Clients and between PrivateSky nodes. zmq_adapter | Node.js binding to the the ZeroMQ. ZeroMQ is used for internal communication inside of PrivateSKy nodes. psk-http-client | Offers http pooling Psklogger | Part of PrivateSky monitoring solution Psk-collector-node | psk-monitoring-ui | psk-smoke-testing | Repositories with various types of tests Psk-integration-testing | Psk-unit-testing | psk-regression | double-check | Library used by tests. Is a simple alternative to Mocha. It is adjusted for and by the PrivateSky team. pskbuffer | Addapts buffer to be serialisable with the isolates technology. csbui | Open source web application for creating new wallets and mobile applications working with SSApps. dicontainer | Experimental dependency injection module soundpubsub | pubsub library to be inside of a single process to safely mediate communication between asynchronous, event driven components. ciengine | Conversational interface experiment/prototype sealjs | Contains adapters to the open source SEAL library created by Microsoft to democratise Homomorphic Encryption usage. acl-magic | Advanced error handlingC library. Reused from SwarmESB, better integration is planned. transrest | Advanced error handling for microservices and swarms. Reused from SwarmESB, better integration is planned. whys | Advanced error handling for microservices and swarms. Reused from SwarmESB, better integration is planned.
