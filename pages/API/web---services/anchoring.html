The Anchoring component of PSK Web Server is linked with the Brick Storage component as it allows the association of a set of brick hashes to a unique identifier called anchor. As explained in the description of the Brick Storage component, a brick is stored on the PSK Web Server in a file whose name is the cryptographic hash of its content. As the data in a brick corresponds to a block of data in a file, any change to that block of data will lead to the creation of a new brick. The Anchoring component allows the grouping of different bricks that correspond to different versions of the same block inside a file under the same anchor.
