This page is short guide for PrivateSky developers, technical writers, external parteners and intribut contributors. The guide is intending to present a brief introduction in how to use the developed web components.

Each page is a simple html writen with a semantic markup defined with web components

Web Components are a set of features that provide a standard component model for the Web allowing for encapsulation and interoperability of individual HTML elements.

Primary technologies used to create them include:

  • Custom Elements – APIs to define new HTML elements
  • Shadow DOM – Encapsulated DOM and styling, with composition
  • HTML Templates, an HTML fragment is not rendered, but stored until it is instantiated via JavaScript
Source:Wikipedia

You can use this starter app in order to build your own app using PrivateSky Custom Web Components.

You can checkout this app and cardinal web components from GitHub.

In order to create a new page, first, we need to understand how works the file named menu.json. Inside this file, there is a configuration that tells the web-site what pages can be displayed.

One of the key names inside the file is "pages". Everything defined inside this key will be displayed in the application menu, so it can be opened. The following example will also show how this page is registered.

{"pages": [{ "name": "Home", "path": "/home", "icon": "fa-home", "pageSrc": "index.html" }, { "name": "About", "path": "/About", "icon": "fa-cog", // The "children" property is defining a sub-tree of pages for the About section. "children": [{ "name": "About", "keyword":"about" }, { "name": "How to create a new site with Cardinal", "keyword":"create-site-cardinal" }, { // As we can see, some pages have the key "path", some do not. // Also, we can see that "pageSrc" attribute is not used for this page. If you use this attribute, you can name the HTML file as you wish. // In this way, the HTML file will be named how-to-write-documentation-pages.html "name": "How to write documentation pages" } ] }]}

Every HTML page is written inside "pages" directory followed by the sub-tree defined in "menu.json". In the above code sample, we have The section About and this page is inside the sub-tree of the section. In this way, the folder structure is the following one: pages/about/how-to-write-documentation-pages.html, so the components will be able to load the page correctly.

So, in order to create a new page, keep in mind that the page needs to be registered in the menu.json file and the HTML file needs to be placed in the folder structure as above explained.

There are three categories of web components so far: main web components, documentation components and form components.

Every component can be used as long as it respects the recommendations of usage.

Check out each component documentation and examples to make an idea of how to use it.
