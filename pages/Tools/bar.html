bar [-option] [-modifier] The bar command allows the storage of files in BAR archives and the manipulation of these archives. -c folderPath
Allows the creation of a BAR archive that stores the files in the directory at folderPath. If a name for the resulting archive was not specified, then, by default the archive will have the same name as the folder at folderPath.

 -f archiveName
This option allows to specify a name for the resulting archive when creating it. It is used in conjunction with the -c option, as follows:

bar -cf folderPath archiveName

The command above creates an archive having the name archiveName that contains the files from the directory at folderPath.

 -x folderPath archiveName
Allows the extraction of files stored by the archive archiveName in the directory at folderPath.

 -t archiveName
Lists all the files contained by the archive archiveName.

 -z
The option above allows the compression of the files when creating an archive. This option is used in conjunction with the option -c, as follows:

bar -czf folderPath archiveName

The above command has as a result the creation of an archive having the name archiveName, that contains the previously compressed files stored in the directory at folderPath. The gzip algorithm is used for compression.
 --key cryptographicKey
Allows the encryption of an archive when creating it. The encryption uses the cryptographicKey argument. The encryption algorithm used is AES in mode GCM. Below is an example that illustrates the usage of this modifier when creating an archive.

bar -czf folderPath archiveName --key=cryptographicKey

 --usage
Prints a short message that shows how to use the bar command.

 --version
Prints the current version of the bar CLI.
